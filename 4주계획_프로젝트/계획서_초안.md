# 실습실 공구 반납 감시대 — 프로젝트 계획서 (초안)

임베디드시스템 4주 프로젝트 · 작성일 2026-07-14

## 1. 개요

실습실 공구 거치대(네트망)를 카메라로 상시 감시하여, YOLO 객체 검출로 공구 재고를 자동 파악하고, RFID 태그로 대여자를 식별한다. 미반납 공구는 시간 초과 시 부저/LED로 경고하고, 태그 없이 공구가 반출되면 즉시 "미확인 반출" 경고를 발생시킨다 (도난 감시).

## 2. 시스템 구성

```
[라즈베리파이]                          [노트북 (서버)]
 ├ Pi 카메라 ──(프레임 HTTP POST)──→  YOLOv8 추론
 ├ RFID RC522 (SPI) ─(태그 UID)──→   재고 대조 + 사용자 귀속 로직
 └ 부저/LED (GPIO) ←─(응답으로 경고 명령)─  Flask 대시보드 + CSV 로그
```

- **라즈베리파이 (엣지)**: 카메라 촬영·전송, RFID 태그 읽기, 부저/LED 경고 구동
- **노트북 (서버)**: YOLO 추론, 재고 판정, 대여자 귀속, 대시보드
- **통신**: HTTP 단일 채널 — Pi가 몇 초마다 프레임을 POST, 응답 JSON에 경고 명령 포함

## 3. 하드웨어

| 품목 | 용도 | 비고 |
|---|---|---|
| Raspberry Pi + Pi 카메라 | 촬영·엣지 제어 | |
| RFID RC522 모듈 + 카드 | 대여자 식별 | SPI 연결, 약 2천원 |
| 부저, LED | 미반납/미확인 반출 경고 | GPIO |
| 네트망 300×300mm | 공구 거치대 | 공구 4개 이하, 간격 30mm 이상 |
| 단색 배경판 (폼보드) | 배경 노이즈 제거 | 검출 정확도·학습 데이터 품질 향상 |
| 공구 4종 | 감시 대상 | 예: 스패너, 드라이버, 펜치, 육각렌치 |

## 4. 소프트웨어 설계

### 4.1 YOLO 검출
- YOLOv8 커스텀 학습 (공구 4종, 종별 100~150장 촬영 + Roboflow 라벨링)
- 학습은 Colab GPU, 추론은 노트북 → Pi 최적화(NCNN 변환 등) 불필요
- 공구판 ROI만 크롭해서 추론 — 배경 오검출 차단

### 4.2 판정 알고리즘
- **개수 기반 판정**: 등록 재고(종류별 개수) − 현재 검출 개수 = 대여 중. 위치 매칭 불필요, 아무 자리에 걸어도 동작
- **디바운스**: 3프레임 연속(약 10초) 동일 판정일 때만 반출/반납 확정 — 오검출·손 가림 방지
- **미반납 경고**: 반출 후 제한 시간(예: 2시간) 초과 시 부저/LED + 대시보드 경고

### 4.3 RFID 대여자 귀속
```
RFID 태그 → 30초 "세션" 열림 → 그 사이 반출/반납된 공구를 해당 UID에 귀속
태그 없이 반출 발생 → 즉시 부저 + "미확인 반출" 경고 (도난 방지)
```
- UID ↔ 이름(학번) 매핑은 사전 등록 (시연용 팀원 등록)

### 4.4 대시보드 · 로그
- Flask 단일 앱: 공구별 상태(재고/대여자/반출 시각), 최근 검출 이미지, 경고 표시
- CSV 로그: 시각, 공구, IN/OUT, 대여자 UID

## 5. 4주 일정

| 주 | 목표 | 산출물 |
|---|---|---|
| 1주 | 거치대 제작(네트망+배경판), 카메라 고정, **데이터 촬영·라벨링·Colab 학습 1차** | 학습된 모델, 검출 데모 |
| 2주 | Pi↔노트북 HTTP 통신, 재고 판정 로직(개수 비교+디바운스), CSV 로그 | 반출/반납 자동 기록 동작 |
| 3주 | RFID 대여자 귀속 + 세션 로직, 부저/LED 경고, Flask 대시보드, 오검출 보며 데이터 추가·재학습 | 전체 기능 통합 |
| 4주 | 통합 테스트, 임계값 튜닝(디바운스·신뢰도·제한시간), 발표자료·시연 영상 | 최종 시연 |

## 6. 리스크 및 대응

| 리스크 | 대응 |
|---|---|
| YOLO 검출 정확도 부족 | 1주차에 학습을 끝내 조기 확인 → 2주차에 데이터 추가·재학습 여유 확보. 비슷한 공구는 클래스 통합 |
| 조명·배경 변화 | 실제 설치 위치·조명에서 데이터 촬영, 단색 배경판, ROI 크롭 |
| 최악의 경우 보험 | 클래스 구분 포기 → "공구 있음" 단일 클래스 + 자리 기반 판별로 후퇴 |
| 네트워크 단절 | Pi가 요청 실패 시 기존 경고 상태 유지 후 재시도 |
