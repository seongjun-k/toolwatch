# YOLO 데이터 수집·라벨링·학습 가이드

공구별 인식 모델을 만드는 전체 절차. 나중에 혼자 보고 따라 할 수 있게 단계별로 정리.

## 전체 흐름

```
수집(서버 수집 모드) → 시드 수동 라벨링(30~50장) → 1차 학습
  → 자동 라벨(tools/autolabel.py) → Roboflow 검수 → 재학습 → best.pt 배치
```

## 0. 사전 준비 (한 번만)

- 기구물·카메라를 **완전히 고정**하고 ROI를 확정한 뒤에 촬영 시작. 세팅을 바꾸면 그동안 찍은 사진은 버리고 재촬영.
- 클래스명을 확정한다. **영문 스네이크케이스, 이후 절대 변경 금지.**
  예: `spanner`, `driver`, `plier`, `hex_key`
  이 이름은 `src/server/config.json`의 `registered_stock` 키와 정확히 일치해야 한다.
- [roboflow.com](https://roboflow.com) 가입 → Create Project → 타입 **Object Detection**.

## 1. 수집 (서버 수집 모드)

1. 서버·Pi를 평소처럼 켠다 (Pi가 3초마다 프레임 전송).
2. 대시보드 → 제어판 → **데이터 수집 모드 시작**. `dataset/raw/날짜_시각/` 폴더에 자동 저장된다.
3. 이 상태에서 아래 변화를 골고루 연출한다:
   - 조명 3종 (시연 시간대 조명 필수)
   - 상태 조합: 전부 걸림 ~ 전부 빠짐, **빈 네트망 필수** (빈자리 오검출 방지)
   - 손 개입·반가림 장면
   - 공구 위치 셔플 (한 자리만 학습하면 위치를 외운다)
4. 끝나면 **수집 중지**. 조명/구성이 바뀔 때마다 새 세션(재시작)으로 나눈다.
5. 폴더를 훑어 거의 똑같은 연속 프레임은 지운다 (1% 크기 필터가 걸러주지만 완벽하지 않음).

## 2. 시드 수동 라벨링 (30~50장)

1. 수집 사진 중 다양한 30~50장을 골라 Roboflow 프로젝트에 업로드.
2. 사진을 열고 공구마다 드래그로 박스를 친 뒤 클래스 선택. 규칙:
   - 박스는 공구 전체를 **타이트하게**.
   - **가려진 공구도 보이는 부분 기준으로 라벨링** (안 하면 손만 대도 사라진 걸로 판정).
   - 빈 네트망 사진은 박스 0개로 그냥 저장 (null 예시로 반드시 포함).
   - 그림자·걸이 고리 포함 여부 같은 애매한 기준은 첫날 정해서 끝까지 통일.
3. Generate 단계:
   - train/valid 분리는 **촬영 세션 단위** (같은 세션 사진이 양쪽에 가면 성능이 뻥튀기됨).
   - 증강은 밝기·노출 ±15~25%만. **회전·크롭·플립 전부 끔** (고정 카메라라 실제로 안 생기는 변형).
4. Export → 포맷 **YOLOv11** → zip 다운로드 → 압축을 풀어 `dataset/` 아래에 배치 (`dataset/data.yaml`이 생기게).

## 3. 1차 학습 (호스트 PC, 저장소 루트)

```
yolo detect train data=dataset/data.yaml model=yolo11n.pt epochs=100 imgsz=640 device=0
```

RTX 3060 기준 10~20분. 결과: `runs/detect/train/weights/best.pt`

## 4. 자동 라벨 → 검수 루프

1. 아직 라벨 안 된 세션 폴더에 자동 라벨 생성 (호스트, 저장소 루트):
   ```
   python tools/autolabel.py dataset/raw/<세션폴더> --model runs/detect/train/weights/best.pt
   ```
   사진 옆에 같은 이름의 `.txt`와 `data.yaml`이 생긴다.
2. 그 폴더(사진+txt+data.yaml)를 통째로 Roboflow에 업로드 → 박스가 이미 그려진 채로 열린다.
3. **틀린 박스만 고친다** (장당 몇 초). 특히 미검출(박스 누락)을 주의해서 볼 것 — 미검출이 유령 OUT 경보로 이어진다.
4. 데이터가 늘었으면 2단계의 Generate/Export를 다시 하고 3단계 학습을 재실행. 모델이 좋아질수록 검수량이 줄어든다.

## 5. 배치·확인

1. `best.pt`를 `model/`에 복사.
2. 클래스명 확인:
   ```
   yolo detect predict model=model/best.pt source=dataset/raw/<아무 세션>/<아무 사진>.jpg
   ```
   출력된 클래스명이 `src/server/config.json`의 `registered_stock` 키와 일치하는지 확인.
3. 서버 재시작 후 대시보드에서 검출 상태 확인.

## 6. 성능이 아쉬울 때 (순서대로)

1. 하이퍼파라미터 만지지 말고 **틀리는 상황 사진 30장 추가** 후 재학습. predict를 돌려 틀린 사진만 모아두면 그게 재학습 목록이 된다.
2. 미검출이 오검출보다 해롭다(유령 OUT → 적색 경보). confidence를 0.5 → 0.35~0.4로 낮추는 방향 먼저.
3. 유령 이벤트가 남으면 `debounce_frames` 3 → 5 (확정 지연 9→15초 대가).
4. 시연 때는 `overdue_sec`을 60초로 단축 (대시보드 제어판에서 변경).
